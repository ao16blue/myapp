3. AWSアカウント作成・初期設定
・AWS Amplifyを起動
・公式を参照し、初期設定
　https://aws.amazon.com/jp/blogs/news/aws-amplify-studio-figma-to-fullstack-react-app-with-minimal-programming/
  - 1で作成したGithubリポジトリと接続
  　※ Privateで作成した場合は、リポジトリ選択画面にある注意書き内に従い、Github側でAWSに接続を許可するリポジトリとして作成したものを選択・設定する必要がある。
  - 基本的には流れに従って操作。最後に「保存してデプロイ」をクリック。
  - アプリ画面で表示される画面メインコンテンツ部分の「Backend environments」タブをクリックし、「Get started」ボタンを押すと、初期設定が走る。（結構時間かかる）
・Backend environments設定のローカル設定手順に基づき、VScodeのターミナルへ入力。ターミナルに表示される指示に従い入力を行う。※この部分、公式で説明省略されているので注意
　(1) 前提条件のページの内容を先に実行
    npm install -g @aws-amplify/cli
    amplify configure
    （ブラウザで設定画面が表示されるのでそれに従いIAMアカウント作成・作成後表示されるアクセスkeyIDとシークレットアクセスKeyが必要。※シークレットアクセスKeyはアカウント作成時しか確認できないので、csvでダウンロードしておく必要あり。紛失・再度環境構築時にアクセスkeyごと新規発行は可能）
  (2) ローカル設定手順に記載のコマンドをターミナルに入力。ターミナルに表示される指示に従い入力を行う。

4. Figmaでラフ作成・連動
・ラフ作成
  - コンテンツなど動的部分は必ずコンポーネントにしておく必要がある。
    ※公式がサンプルっ公開しているFigmaのUI Kitを参照するとわかりやすい（https://www.figma.com/community/file/1047600760128127424）
  - コンポーネントごとにAWS側でDBのフィールドと連携させるため、名称はそれぞれユニークである必要アリ。
  - AWS側では色設定・文字サイズの変更などの機能なしだが、Figma側の変更が（手動読み込みにて）反映されるので、後からの修正も可能。
・AWSとFigma連携
  - AWS AmplifyのBackend environmentsの「Studioを起動する」から開発画面へ。
  - サイドナビ Designの「UI Library」をクリック
  - 「Sync Width Figma」ボタンを押して、Figma側のラフで「共有」ボタンから発行されるURLをコピー＆ペースト
